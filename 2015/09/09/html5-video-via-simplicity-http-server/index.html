HTML5 video via Simplicity HTTP server

09 September 2015

tl;dr

HTML5 video 再生のテストには WEBrick がいいかも？

ある LP の HTML5 video 再生をテストしていました。

Python の SimpleHTTPServer, PHP の Built-in web server そして Ruby の WEBrick を試して、 Safari から正常に再生できたのは WEBrick のみでした。

各環境とコマンドラインはこんな感じです。

Safari

navigator.userAgent
"Mozilla/5.0 (Macintosh; Intel Mac OS X 10_11) AppleWebKit/601.1.37 (KHTML, like Gecko) Version/9.0 Safari/601.1.37" = $1

Python

python -V
Python 2.7.10

python -m SimpleHTTPServer 9999

PHP

php -v
PHP 5.5.24 (cli) (built: Jun 26 2015 18:17:36)
Copyright (c) 1997-2015 The PHP Group
Zend Engine v2.5.0, Copyright (c) 1998-2015 Zend Technologies

php -S 0.0.0.0:9999 -t .

Ruby

ruby -v
ruby 2.2.3p173 (2015-08-18 revision 51636) [x86_64-darwin15]

ruby -rwebrick -e 's = WEBrick::HTTPServer.new(BindAddress: "0.0.0.0", Port: 9999, DocumentRoot: Dir.pwd);trap("INT"){s.shutdown};s.start'

当初ちっとも再生されなくて、「作成した MP4 の形式が不整合を起こしているのかも」等と、無駄な時間を使いました。

ご注意

WEBrick 以外もコマンドラインオプション等々で回避できそうだけど追ってません。

そもそも Ruby 以外は最新リリースですらない、というのも不公平（？）ですので、できたら追加検証するかもしれません。（きっとやらない）
