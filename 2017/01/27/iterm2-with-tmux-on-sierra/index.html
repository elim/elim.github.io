Sierra 上の iTerm2 上の tmux

Posted on January 27, 2017

ここの設定を変えると幸せになれます。

iTerm2 + tmux で 2 つ目のタブをつくると最初のタブの描画がこわれるの、 2 つ目のタブができた時点でタブバーが表示されて、描画域がかわるからだった。どうも tmux がそれに追従できなくて 1 行たらない状態になるみたい。なぜ Sierra だけで起きるんだろう。

— lint厨👮🏻 (@elim) January 27, 2017

とりあえずの解決策としては、"Preferences" => "Appearance" => "Show tab bar even when there is only one tab" をオンにして、常にタブバーを表示すること。

— lint厨👮🏻 (@elim) January 27, 2017
